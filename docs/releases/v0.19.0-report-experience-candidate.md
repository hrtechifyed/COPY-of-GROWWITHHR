# v0.19.0 Report Experience Candidate

This document records the candidate scope before a formal version tag is created.

## Candidate features

- Light and dark HRTechify PDF report styles selected at lead capture.
- Compact A4 report presentation with the established logo.
- One Person Company workforce count locked at one.
- Minimum employee count of one for other entity types.
- Count-aware workforce grammar.
- Deduplicated web/PDF presentation.
- Actionable recommendations with how-to steps, ownership, timing and downloadable templates.

## Protected behavior

- Public page and navigation layout remain unchanged.
- Assessment chapters and storage keys remain unchanged.
- Gmail delivery and origin-restricted Render API boundaries remain unchanged.
- Compliance applicability and private-beta M1–M3 models remain unchanged.

## Required validation before release

- `npm run test:report-experience`
- `npm run test:release`
- `npm run test:release:e2e`
- manual One Person Company journey;
- manual one-employee and multi-employee grammar review;
- manual light PDF download and email attachment review;
- manual dark PDF download and email attachment review;
- website report and template-link review;
- GitHub Pages and Render delivery smoke test.

A formal `v0.19.0` tag must not be created until the complete release gate passes and a rollback commit is recorded.
