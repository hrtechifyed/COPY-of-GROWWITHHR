## Purpose

M2 introduces a deterministic recommendation-traceability foundation for the GrowWithHR Compliance DNA private-beta experience.

The milestone explains:

- which recommendation was produced;
- which confirmed facts were supplied by the user;
- which facts were derived deterministically;
- which versioned rule was evaluated;
- which applicability status was assigned;
- which required information remains missing;
- which evidence status applies;
- which action is suggested;
- which structured sources support further review;
- which limitations apply to the advisory output.

M2 does not replace the stable Executive Advisory Briefing.

It does not modify the protected stable report, PDF, email or delivery contracts.

## Advisory boundary

GrowWithHR remains an advisory product.

M2 output must not be presented as:

- legal certification;
- verified legal compliance;
- verified non-compliance;
- government approval;
- completed registration;
- completed filing;
- evidence verification;

## Record separation

M2 keeps the following concepts separate.

### Confirmed facts

Confirmed facts record answers explicitly supplied through the protected assessment contract.

A confirmed fact does not prove that an obligation has been completed.

### Derived facts

Derived facts are calculated deterministically from confirmed facts.

Every derived fact records the source fact identifiers used to create it.

### Rule evaluations

A rule evaluation records:

- the rule identifier;
- the rule version;
- required fact identifiers;
- triggering fact identifiers;
- missing fact identifiers;
- applicability status;
- reason;
- source references;
- evaluation timestamp.

### Recommendations

A recommendation records:

- a stable recommendation identifier;
- the originating rule identifier;
- applicability status;
- title;
- reason;
- suggested action;
- suggested timeline;
- triggering facts;
- missing information;
- evidence status;
- source references;
- limitations.

### Evidence status

Evidence status describes supporting material.

It does not determine rule applicability.

### Source references

Source references identify official or authoritative materials for further review.

A source reference does not prove that a rule applies to the assessed organisation.

## Applicability statuses

M2 supports:

- `applicable`
- `likely-applicable`
- `not-currently-applicable`
- `more-information-needed`
- `specialist-review`

### `applicable`

The available facts satisfy the deterministic rule conditions.

This is an advisory rule result, not verified legal applicability.

### `likely-applicable`

The available facts strongly indicate applicability, but confirmation remains necessary.

### `not-currently-applicable`

The available facts do not satisfy the current deterministic conditions.

This status must not be used merely because required information is missing.

### `more-information-needed`

One or more required facts are absent, unclear or unsupported.

The evaluator must identify the missing fact identifiers.

### `specialist-review`

The rule requires professional interpretation, jurisdiction-specific confirmation or facts outside the current assessment contract.

## Evidence statuses

M2 supports:

- `not-requested`
- `not-provided`
- `provided`
- `not-verified`
- `verified`

The initial recommendation catalog defaults to:

`not-requested`

The current assessment does not collect or verify documentary evidence.

The status `verified` must not be assigned without an explicitly approved verification process.

1. Keep `complianceDnaV3` set to `false`.
2. Continue routing public assessment traffic to `/analyze-company.html`.
3. Revert the private-beta diagnostics interface and controller commits when required.
4. Retain the protected v2 assessment, report, lead and delivery records.
5. Do not delete or reset protected browser storage.
6. Confirm the stable assessment, report, PDF and email paths remain operational.

Because M2 does not create a browser-storage key, no traceability storage cleanup is required.
